# Attention 机制笔记

> 基于论文 Attention Is All You Need (1706.03762)

---

## 前置知识

### 向量是什么

向量是一组有顺序的数字，可以理解为空间里的一个箭头，有方向和长度。

```
a = [3, 2]  →  从原点出发，向右3步，向上2步
```

### 点积

两个向量对应位置相乘，再全部加起来，结果是一个数。

```
a = [3, 2],  b = [1, 4]
a · b = 3×1 + 2×4 = 11
```

**几何意义：点积越大 = 两向量方向越接近 = 越相似。**

| 两向量关系 | 点积 |
|-----------|------|
| 方向完全一致 | 最大正数 |
| 方向垂直 | 0 |
| 方向完全相反 | 最小负数 |

### 词向量 (Embedding)

把词映射成高维向量（512维），语义相近的词向量方向相近。

```
"猫"  → [0.9, 0.1, 0.8, ...]
"狗"  → [0.8, 0.2, 0.7, ...]  ← 和猫接近
"苹果"→ [0.1, 0.9, -0.3, ...] ← 和猫差远
```

本质是一张查找表（50000×512的矩阵），随机初始化，训练时自动调整。

### 线性变换

向量乘以矩阵，得到另一个向量。直觉：**换一个角度看同一个东西。**

```
x（512维） × W（512×64） = 新向量（64维）
```

矩阵 W 是训练出来的，模型自己学会"什么角度最有用"。

### Softmax

把一组任意数字变成概率分布（每个值在0~1之间，加起来=1）。

```
softmax([2.0, 1.0, 0.1])

e^2.0=7.39, e^1.0=2.72, e^0.1=1.10, 总和=11.21

→ [0.659, 0.242, 0.099]
```

用指数的原因：放大差距，让模型更果断地关注重要的词。

## Q、K、V 是什么

同一个词向量，乘以三个不同的矩阵，得到三个角色：

| 角色 | 全称 | 含义 | 类比（搜索引擎） |
|------|------|------|----------------|
| Q | Query（查询） | 我想找什么 | 搜索词 |
| K | Key（键） | 我能被谁找到 | 网页标题/标签 |
| V | Value（值） | 我能提供什么信息 | 网页正文 |

```
x（词向量，512维）
├── × WQ（512×64） → Q（64维）
├── × WK（512×64） → K（64维）
└── × WV（512×64） → V（64维）
```

WQ、WK、WV 是三个独立的可学习矩阵，训练时自动调整。

**K 和 V 为什么分开？**
K 负责"被找到"，V 负责"传递信息"，分开让模型有更多灵活性。

## Scaled Dot-Product Attention

### 公式

```
Attention(Q, K, V) = softmax( Q·Kᵀ / √dk ) · V
```

### 完整数值例子

输入句子：`"猫 吃 鱼"`，简化为 4维词向量、dk=dv=3维。

**原始词向量（Embedding）：**

```
"猫" = x₁ = [1, 0, 1, 0]
"吃" = x₂ = [0, 1, 0, 1]
"鱼" = x₃ = [1, 1, 0, 0]
```

**线性变换，得到 Q、K、V（乘以训练好的矩阵）：**

```
Q（查询矩阵）:        K（键矩阵）:          V（值矩阵）:
猫: Q₁ = [2, 0, 1]   猫: K₁ = [0, 1, 1]   猫: V₁ = [1, 0, 1]
吃: Q₂ = [0, 1, 1]   吃: K₂ = [1, 1, 0]   吃: V₂ = [0, 1, 0]
鱼: Q₃ = [1, 1, 0]   鱼: K₃ = [1, 0, 1]   鱼: V₃ = [1, 1, 0]
```

### 逐步拆解

**步骤1：Q·Kᵀ — 算相似度**

每个词的 Q 和所有词的 K 做点积，得到 3×3 相似度矩阵。

```
Q₁·K₁ = 2×0+0×1+1×1 = 1   Q₁·K₂ = 2×1+0×1+1×0 = 2   Q₁·K₃ = 2×1+0×0+1×1 = 3
Q₂·K₁ = 0×0+1×1+1×1 = 2   Q₂·K₂ = 0×1+1×1+1×0 = 1   Q₂·K₃ = 0×1+1×0+1×1 = 1
Q₃·K₁ = 1×0+1×1+0×1 = 1   Q₃·K₂ = 1×1+1×1+0×0 = 2   Q₃·K₃ = 1×1+1×0+0×1 = 1

相似度矩阵（行=当前词，列=被关注的词）:
         猫  吃  鱼
猫   [[ 1,  2,  3 ],
吃    [ 2,  1,  1 ],
鱼    [ 1,  2,  1 ]]
```

第 i 行第 j 列 = 词 i 在理解自己时，应该从词 j 借多少信息。

**步骤2：÷√dk — 缩放**

dk=3，√3≈1.73，每个数除以 1.73：

```
         猫     吃     鱼
猫   [[ 0.58, 1.15, 1.73 ],
吃    [ 1.15, 0.58, 0.58 ],
鱼    [ 0.58, 1.15, 0.58 ]]
```

dk 是 K 向量的维度（论文中=64）。维度大时点积数值会很大，导致 softmax 梯度极小（梯度消失），训练困难。除以 √dk 压回合理范围。

**步骤3：softmax — 变成注意力权重**

对每一行做 softmax，变成概率分布：

```
"猫"行 [0.58, 1.15, 1.73]:
  e^0.58=1.79, e^1.15=3.16, e^1.73=5.64, 总和=10.59
  → [0.17, 0.30, 0.53]

"吃"行 [1.15, 0.58, 0.58]:
  e^1.15=3.16, e^0.58=1.79, e^0.58=1.79, 总和=6.74
  → [0.47, 0.27, 0.27]

"鱼"行 [0.58, 1.15, 0.58]:
  → [0.21, 0.57, 0.21]

注意力权重矩阵:
         猫     吃     鱼
猫   [[ 0.17, 0.30, 0.53 ],   ← 猫最关注鱼
吃    [ 0.47, 0.27, 0.27 ],   ← 吃最关注猫
鱼    [ 0.21, 0.57, 0.21 ]]   ← 鱼最关注吃
```

每行和=1，代表当前词对所有词的注意力分布。

**步骤4：×V — 加权求和**

按注意力权重，把所有词的 V 混合起来：

```
新猫 = 0.17×V₁ + 0.30×V₂ + 0.53×V₃
     = 0.17×[1,0,1] + 0.30×[0,1,0] + 0.53×[1,1,0]
     = [0.17,0,0.17] + [0,0.30,0] + [0.53,0.53,0]
     = [0.70, 0.83, 0.17]
```

原来"猫"= [1, 0, 1]，现在变成了 [0.70, 0.83, 0.17]，融合了"吃"（权重0.30）和"鱼"（权重0.53）的信息。

### 结果

每个词得到一个新向量，融合了整个句子的上下文信息：

```
"猫吃鱼" 里的"猫" → 融合了"吃"和"鱼"的信息 → "正在吃鱼的猫"
"猫追狗" 里的"猫" → 融合了"追"和"狗"的信息 → "正在追狗的猫"
```

同一个词，在不同句子里，经过 Attention 后得到不同的向量表示。

## Multi-Head Attention

### 为什么需要多头

单头 attention 只能从一个角度看词与词的关系，输出是所有关系的混合，难以同时精确捕捉多种语义关系。

### 做法

把整个 Attention 流程并行跑 h=8 次，每次用不同的 WQ/WK/WV：

```
原始词向量（512维）
    ↓
head1: ×W1Q/W1K/W1V → Attention → 64维输出
head2: ×W2Q/W2K/W2V → Attention → 64维输出
...
head8: ×W8Q/W8K/W8V → Attention → 64维输出
    ↓
Concat → 512维（64×8）
    ↓
× WO（512×512）→ 最终输出（512维）
```

### 公式

```
MultiHead(Q, K, V) = Concat(head₁, ..., head₈) · WO

其中 headᵢ = Attention(Q·WiQ, K·WiK, V·WiV)
```

### 每个头学到什么

每组矩阵独立训练，自动学到不同类型的关系：

```
head 1 → 主语-谓语关系
head 2 → 代词-指代关系
head 3 → 修饰-被修饰关系
...（训练自动涌现，非人工规定）
```

### 数值例子

还是用"猫吃鱼"，简化为 2个头（h=2），每个头 dk=dv=2维。

**两个头用不同的矩阵，对同样的输入做 attention：**

```
原始词向量（简化4维）:
猫: x₁=[1,0,1,0]   吃: x₂=[0,1,0,1]   鱼: x₃=[1,1,0,0]
```

**head 1（假设学到了"动作关系"）：**

```
用 W1Q/W1K/W1V 投影到2维后，做完整 attention，得到注意力权重：
         猫     吃     鱼
猫   [[ 0.10, 0.70, 0.20 ],   ← 猫最关注"吃"（动作）
吃    [ 0.20, 0.20, 0.60 ],   ← 吃最关注"鱼"（对象）
鱼    [ 0.10, 0.60, 0.30 ]]

head1 输出"猫"的向量: [0.62, 0.38]  （2维）
```

**head 2（假设学到了"同类关系"）：**

```
用 W2Q/W2K/W2V 投影到2维后，做完整 attention，得到注意力权重：
         猫     吃     鱼
猫   [[ 0.80, 0.10, 0.10 ],   ← 猫最关注自己（名词聚焦）
吃    [ 0.10, 0.80, 0.10 ],
鱼    [ 0.15, 0.10, 0.75 ]]

head2 输出"猫"的向量: [0.71, 0.29]  （2维）
```

**两个头的输出拼接：**

```
head1("猫") = [0.62, 0.38]   （2维）
head2("猫") = [0.71, 0.29]   （2维）

Concat → [0.62, 0.38, 0.71, 0.29]   （4维）
```

**乘以 WO 做最终融合：**

```
[0.62, 0.38, 0.71, 0.29] × WO（4×4） = 最终"猫"的向量（4维）
```

**关键点**：两个头的注意力分布完全不同——head1 让"猫"关注"吃"，head2 让"猫"关注自己。最终向量同时包含了两种信息，这是单头做不到的。

### 计算量没有增加

| | 投影参数量 |
|--|-----------|
| 单头（512维） | 3 × (512×512) = 786432 |
| 多头（8头×64维） | 8 × 3 × (512×64) = 786432 |

参数量相同，但表达能力强得多。

## Attention 在 Transformer 中的三种用法

| 位置 | Q 来自 | K/V 来自 | 作用 |
|------|--------|----------|------|
| Encoder Self-Attention | 输入序列 | 输入序列 | 每个词看整个输入句子 |
| Decoder Masked Self-Attention | 输出序列 | 输出序列（masked） | 生成时只能看已生成的词 |
| Encoder-Decoder Cross-Attention | Decoder | Encoder输出 | Decoder 看整个输入序列 |

**Mask 的作用**：Decoder 生成第 i 个词时，不能看到第 i+1 之后的词。实现方式：把未来位置的相似度设为 -∞，softmax 后权重变成 0。

## 完整流程图

```
输入句子: "猫 吃 鱼"
    ↓ Embedding
每个词 → 512维向量
    ↓ + Positional Encoding（加位置信息）
带位置信息的词向量
    ↓ × WQ, WK, WV（线性变换）
Q, K, V 矩阵
    ↓ Q·Kᵀ（点积，算相似度）
n×n 相似度矩阵
    ↓ ÷ √dk（缩放）
    ↓ softmax（变成概率）
注意力权重矩阵
    ↓ × V（加权求和）
单头输出
    ↓ 8个头并行 + Concat + WO
Multi-Head Attention 输出
```

---

## 一句话总结

> Attention 让每个词通过点积找到句子里和自己相关的词，按相关程度混合它们的信息，从而理解自己在当前语境下的具体含义。Multi-Head 则从 8 个角度同时做这件事，捕捉多种语义关系。
